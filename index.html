Transitions and Animations

Transitions vs. Animations

Transitions

  • only targets the start and end point
  • runs once
  • requires some form of trigger to run (ie. hover or focus)

Animations

  • can target multiple points within the animation's timeline
  • runs as many times as specified
  • runs automatically (although can alter its behaviour to wait for a trigger)
  • both can affect more than one CSS property

Shared CSS Properties

  • -duration

    The time it takes in seconds or miliseconds from the beginning to the end of the transition / animation.

  • -delay

    The delay in seconds or miliseconds before the transition / animation begins

    hover here

  • -timing-function

    The relative speed of the animation as it moves through its lifecycle.

    • linear

    • ease

    • ease-out

    • ease-in

    • ease-in-out

    • cubic-bezier

  • What is a cubic bezier?

    • takes 4 values, representing the location of the two control points (P1 and P2)
    • first and third values are the x coordinates, with a value within the range of 0-1
    • second and fourth values are the y coordinates
    • the steeper the slope, the faster the animation / transition moves

  • Property

    Both animations and transitions require the dev to specify which CSS properties are being targeted, although the syntax for transitions and animations are different. Common properties include opacity, position (top/right/bottom/left), width and height, to name a few.

    Transform

    The CSS transform property provides more opportunities for animations / transitions, can 2D or 3D transformations.

    • Scale()

      • Makes an element larger or smaller to its original size
      • scale(2) makes an element 2x bigger, scale(0.5) makes it half as big

    • Translate(a,b) / TranslateX(a) / TranslateY(b)

      • Moves an element up/down/left/right by a certain % or pixels
      • Negative X values will move the element left, and Y values, up

    • Rotate() / RotateX() / RotateY()

      • Rotates the element in either a 2D or 3D plane
      • To sepcify the degree of rotation, include a number followed by 'deg' (ie. rotate(45deg))
      • Using rotateX() or rotateY() will rotate the element around the X or Y axis
    • Other examples include skew and persepctive

A Bit More About Animations

Animations are more complex than transitions, and offer additional CSS properties that offer greater control and complexity

  • -iteration-count

    The number of times the animation should run. The default is once, but can be as high as the keyword infinite

  • -fill-mode

    It determines the styles once the animation is complete (will it revert back to its original state, or keep the styles applied at at the end of the animation

    • none: default (will revert back to original state
    • forwards: will retrain styles from the last keyframe
    • backwards: will retain its style from the first keyframe
    • both: will follow the rules for both forward and backwards

  • -direction

    Tells the animation the direction in which it should run

    • normal: default (forwards)
    • reverse: backwards
    • alternate: alternate back and forth
    • alternate-reverse: alternate, but start backwards then run forwards

  • -play-state

    Just like in transitions, you can force the animation to wait for an event to trigger it (such as a hover / focus).

Accessibility

While animations can add interest, some users can find animations dizzying or distracting, and prefer to opt out of them in browser preferences.

The media query @prefers-reduced-motion can be used to tailor animations or transitions for users that want either no movement or very little.
